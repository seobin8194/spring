1강 스프링의 개요
1-1 스프링 프레임워크 모듈
스프링 legacy                                                                 스프링 boot
original, custom이 자유롭다(직접 설정하는게 많다)                 legacy에서 수작업했던것을 자동으로

- 스프링 프레임워크의 주요기능 : DI MVC AOP JDBC
* 프레임워크
기본 뼈대를 이루는 코드의 묶음
방향성을 제시하고 원하는 기능을 빨리 만들 수 있다
* 모듈 혹은 라이브러리
프레임워크안에 미리 만들어져 있는 기능

- 스프링 특징
1. POJO(Plain Java Object)
스프링 컨테이너로부터 자바 객체를 얻을 수 있는데 이때 원래 순수한 객체를 이용한다
즉 기능이 너무 많아 규약이 많은 객체가 아닌 자기기능만하는 순수한 객체를 이용한다
2. DI(Dependency Injection)
객체 사이에 의존성이 존재할 경우 스프링이 미리 결합해놓는다
클래스 사이의 결합 약화 
-> a객체 생성시 b객체를 생성해야하는 경우 스프링이 a와 b를 미리 결합하여 a를 생성하면 b도 생성되게
3. AOP(Aspect Oriented Programming)
트랜잭션, 로깅, 보안 등 여러 모듈에서 공통적으로 지원하는 기능을 분리하여 사용
이는 test와 개발을 동시에 할 수 있게 한다 -> 개발 중 기능만 테스트해 볼 수 있다
4. 확장성
스프링은 소스를 모두 라이브러리로 분리하여 필요한 라이브러리만 가져다 쓸 수 있다
외부 라이브러리와도 연동하여 사용할 수 있다
5. mvc 지원
m : data 관련 v : view c : 화면 응답 요청 s(service) : 잡일

- 스프링 모듈
spring-core : DI와 IOC 기능제공
spring-aop : AOP 기능 제공
spring-jdbc : 데이터베이스를 다룰 수 있는 기능
spring-tx : 트랜젝션관련 기능
spring-webmvc : MVC 구현 기능

1-2 스프링 컨테이너 (IOC)
객체를 사용전 미리 생성하고 조립하는 컨테이너
-> 제어의 역전 (Inversion of Control) : 객체가 필요할때 생성해서 사용하던 기존 방식과 반대로 미리 생성하고 필요할때 꺼내서 사용
컨테이너를 통해 생성된 객체를 빈(Bean)이라고 한다
